Explain Graph Neural Network Expressive Power with Combinatorial Problem

Approximation Ratios of Graph Neural Networks for Combinatorial Problems

In this section we explore the expressiveness of Graph Neural Networks (GNNs) from yet another perspective. This time, we look at how well GNNs can learn to express the solutions to combinatorial problems. This may seem odd as combinatorial problems belong to \( NP \) and thus cannot be solved by GNNs, which run in polynomial time. However, using polynomial-time algorithms to approximate solutions to combinatorial problems is a well-studied area with established methods for quantifying how well-suited an algorithm is to serve as an approximation. By applying these ideas to the class of algorithms that are learnable by GNNs, we can better understand the limits of GNN models and even devise architectures which have superior approximation ability.

Makes analogy between GNNs and distributed local algorithms, which solve problems in constant time by splitting them amongst nodes that we assume have infinite computation but these nodes can only communicate with each other a constant () number of times. This analogy is then used to show that GNN architectures correspond to various distributed local algorithms and their ability to solve combinatorial problems is determined by the expressivity of the algorithm to which they correspond.
